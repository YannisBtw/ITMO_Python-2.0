Task Tracker

Учебный таск-трекер (TODO App) на Django: регистрация/авторизация, полный CRUD по задачам и пользователям, SQLite и Bootstrap 5.

Видеодемонстрация сайта

Команда

  • Команда "Кимпинтяо"
  • Участники: Ломаченко Ян (Controllers/Routing/Admin), Брискиндов Олег (Backend/ORM), Шанин Иван (Frontend).

Стек и архитектура

  • Python 3.13, Django 6.0, база данных SQLite.
  • Кастомная модель пользователя с логином по email и обязательным полным именем.
  • Django Templates + Bootstrap 5; MVT-архитектура (соответствует требованию MVC).
  • Приложение tracker содержит модели, формы, контроллеры, вьюхи, статику и шаблоны; конфигурация проекта - в tasktracker/, базовый шаблон - templates/base.html.

Функциональность

  • Регистрация, вход и выход; авторизация обязательна для работы с задачами.
  • Дашборд со своими задачами: создание, редактирование, удаление.
  • Поля задачи: название, описание, приоритет (low/medium/high), статус (todo/in_progress/done); изменение статуса отдельной формой.
  • Просмотр всех задач системы и списка пользователей с количеством их задач.
  • Удаление собственного аккаунта.
  • Админ-панель Django (/admin/) с полным CRUD по пользователям и задачам.

Быстрый запуск (Windows / PowerShell)

cd lab_11
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install "django>=6.0,<6.1"
python manage.py migrate
python manage.py createsuperuser  # админ для проверки CRUD всех сущностей
python manage.py runserver

Откройте http://127.0.0.1:8000 и зарегистрируйтесь или войдите под суперпользователем. Для чистого старта удалите db.sqlite3 перед миграциями.

Полезные маршруты

  • / - дашборд текущего пользователя.
  • /register/, /login/, /logout/.
  • /tasks/ - все задачи; /tasks/new/, /tasks/<id>/edit/, /tasks/<id>/delete/, /tasks/<id>/status/.
  • /users/ - список зарегистрированных пользователей и количество их задач.
  • /account/delete/ - удаление собственного профиля.
  • /admin/ - административный интерфейс.
