Тема лабораторной работы

Создание и развертывание статического сайта на базе MkDocs с публикацией на GitHub Pages

Цель работы

Освоить процесс создания статического сайта с использованием генератора документации MkDocs, а также изучить основные принципы публикации сайта с помощью GitHub Pages.

В ходе выполнения лабораторной работы были освоены:

  • создание статического сайта из Markdown-файлов;
  • организация структуры документации проекта;
  • использование системы контроля версий Git;
  • работа с платформой GitHub;
  • публикация сайта в интернете через GitHub Pages;
  • настройка конфигурационного файла mkdocs.yml.

Используемые технологии

В работе использовались следующие инструменты:

  • Python
  • MkDocs — генератор статических сайтов
  • Git — система контроля версий
  • GitHub — платформа для хранения и публикации проекта
  • GitHub Pages — сервис для публикации статических сайтов

Локальный запуск проекта

Для локальной разработки используется встроенный сервер MkDocs.

1. Клонирование репозитория

git clone https://github.com/YannisBtw/yannisbtw.github.io
cd yannisbtw.github.io

2. Создание виртуального окружения

python -m venv .venv

3. Активация окружения

Windows:

.venv\Scripts\activate

4. Установка MkDocs

pip install mkdocs

5. Запуск локального сервера

mkdocs serve -f site/mkdocs.yml

После запуска сайт будет доступен по адресу:

http://127.0.0.1:8000

Изменения в Markdown файлах отображаются автоматически.

Сборка сайта

Для публикации сайта используется команда:

mkdocs build -f site/mkdocs.yml -d ../docs --clean

После сборки в папке docs появляется готовый статический сайт.

Публикация сайта

Публикация осуществляется через GitHub Pages.

Настройки:

  • ветка: main
  • папка: /docs

После отправки изменений в репозиторий сайт автоматически обновляется.
